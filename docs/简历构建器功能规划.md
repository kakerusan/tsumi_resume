# 简历构建器功能规划文档

## 1. 产品目标

构建一个类似截图所示的在线简历构建窗口，采用「左侧编辑区 + 右侧实时预览区」的双栏模式，让用户可以一边填写内容，一边看到最终简历效果。

核心目标如下：

1. 支持快速录入常见简历信息，并尽量减少排版成本。
2. 支持重点字段、关键词、技术名词的加粗展示，突出个人优势。
3. 支持在实习经历中展示公司 Logo，增强真实感与视觉识别度。
4. 界面要美观、专业、适合导出打印，整体风格接近正式校招/社招简历。
5. 功能尽量全面，但开发应分阶段推进，优先保证可用性和稳定性。

## 2. 对标页面的关键特征

从截图可以抽象出以下核心设计：

1. 左侧为表单式内容编辑器，按模块折叠管理。
2. 右侧为 A4 风格简历预览，支持实时渲染。
3. 顶部有示例、清空、保存、导出等快捷操作。
4. 每个模块可以单独展开、收起、排序、增删。
5. 简历内容强调层次感：
   个人信息明显
   模块标题统一
   关键词加粗
   技术标签高亮
6. 页面兼顾“编辑效率”和“最终成品观感”。

## 3. 建议的页面结构

建议整体分成以下区域：

### 3.1 顶部操作栏

建议功能：

1. 加载示例数据
2. 清空当前表单
3. 本地保存
4. 导出 PDF
5. 导出图片
6. 导入 JSON
7. 恢复上次编辑内容

### 3.2 左侧编辑面板

建议采用卡片式折叠分组：

1. 基本信息
2. 教育背景
3. 相关技能
4. 实习经历
5. 项目经历
6. 荣誉奖项
7. 证书语言
8. 个人总结 / 自我评价
9.  主题配置
10. 模块顺序管理

### 3.3 右侧预览面板

建议特点：

1. 固定为 A4 比例预览
2. 与编辑内容实时同步
3. 支持缩放
4. 支持分页提示
5. 导出时保持和预览一致

## 4. 功能需求清单

下面按模块拆分需求，便于后续直接开发。

### 4.1 基本信息模块

字段建议：

1. 姓名
2. 求职意向
3. 电话
4. 邮箱
5. GitHub / Gitee / 个人网站
6. 所在城市
7. 学校
8. 专业
9. 学历
10. 起止时间
11. GPA
12. 英语成绩 / 证书成绩
13. 头像 / 证件照

交互要求：

1. 支持头像上传、替换、删除
2. 联系方式支持图标展示
3. 可配置“教育信息是否放在姓名下方”
4. 长链接自动省略显示，但导出时保留完整信息

### 4.2 教育背景模块

字段建议：

1. 学校名称
2. 学历
3. 专业
4. 起止时间
5. 主修课程
6. GPA
7. 排名
8. 亮点补充

交互要求：

1. 支持多段教育经历
2. 支持拖拽排序
3. 可设置“仅显示最高学历”

### 4.3 技能模块

这是你非常需要做好的一个模块，因为它直接决定简历是否有“重点感”。

字段建议：

1. 技能分类名
2. 技能描述
3. 技能标签列表
4. 熟练度

必须支持的能力：

1. 关键词加粗
2. 支持多条技能分组
3. 支持标签化展示，例如 `SpringBoot`、`Redis`、`MySQL`
4. 支持控制描述顺序

关键词加粗建议做法：

1. 基础方案：用户输入 Markdown 风格内容，例如 `**SpringBoot**`
2. 进阶方案：提供“选中文本后加粗”的编辑按钮
3. 高级方案：维护关键词词库，自动识别并高亮技术名词

建议优先级：

1. 第一版先支持手动加粗
2. 第二版再做富文本工具条
3. 第三版再做关键词自动识别

### 4.4 实习经历模块

这是你的重点模块之一，建议做成“信息条 + 内容描述”的结构。

字段建议：

1. 公司名称
2. 公司 Logo
3. 部门 / 业务线
4. 岗位名称
5. 所在城市
6. 起止时间
7. 工作简介
8. 工作内容列表
9. 工作成果列表
10. 关联技术栈

必须支持的能力：

1. 展示公司 Logo
2. 支持上传、替换、删除 Logo
3. 支持 Logo 在左侧、右侧、行尾三种位置
4. 支持无 Logo 时自动退化为纯文字布局
5. 支持多段实习经历
6. 支持排序
7. 支持关键成果加粗

Logo 功能详细要求：

1. 支持本地图片上传，格式至少支持 PNG、JPG、SVG
2. 上传后可裁剪或限制统一比例
3. 预览区统一控制 Logo 大小，避免视觉混乱
4. 支持白底 Logo 与透明 Logo
5. 导出 PDF 时保证 Logo 清晰

建议的显示方式：

1. 公司名称 + 岗位名称作为主标题
2. 时间、地点、部门作为副信息
3. Logo 放在信息条右侧最稳妥

### 4.5 项目经历模块

字段建议：

1. 项目名称
2. 项目副标题
3. 项目时间
4. 项目角色
5. 项目简介
6. 技术栈标签
7. 核心功能点
8. 难点与优化点
9. 项目链接
10. 仓库链接

必须支持的能力：

1. 技术栈标签展示
2. 核心成果加粗
3. 长项目支持自动分页
4. 项目链接可选显示

### 4.6 荣誉奖项 / 证书模块

字段建议：

1. 奖项名称
2. 获奖时间
3. 颁发单位
4. 奖项等级
5. 补充描述

交互要求：

1. 支持列表模式
2. 支持时间排序
3. 可隐藏细节，仅保留精简展示

### 4.7 自我评价模块

字段建议：

1. 标题
2. 内容
3. 关键词

交互要求：

1. 支持限制字数
2. 支持一键加粗关键词
3. 支持切换显示 / 隐藏

## 5. 全局能力要求

### 5.1 模块管理

建议所有模块都支持：

1. 展开 / 收起
2. 显示 / 隐藏
3. 拖拽排序
4. 新增
5. 删除
6. 复制

### 5.2 富文本与排版能力

这是整个产品体验的核心能力之一。

建议至少支持：

1. 加粗
2. 斜体
3. 下划线
4. 项目符号列表
5. 自定义行距
6. 段前段后间距
7. 技术标签样式
8. 标题层级样式

加粗功能建议覆盖以下场景：

1. 技术栈关键词
2. 实习成果指标
3. 项目亮点
4. 获奖关键词
5. 姓名、学校、岗位、公司等头部关键信息

### 5.3 主题与美观能力

你提到“美观”，这部分不能只靠颜色，应该做成完整的主题系统。

建议支持：

1. 主色切换
2. 标题样式切换
3. 字体组合切换
4. 页面边距切换
5. 模块间距切换
6. 标签样式切换
7. 时间轴样式切换
8. 头像形状切换
9. 是否显示分隔线

视觉建议：

1. 默认模板风格要正式、克制、适合打印
2. 标题建议使用明显但不过度夸张的蓝色或深色体系
3. 正文字体要保证打印清晰
4. 关键词的加粗颜色不要过多，避免杂乱
5. Logo、标签、图标要统一视觉尺寸

推荐做 3 套模板：

1. 校招正式版
2. 技术极简版
3. 设计感增强版

### 5.4 导出与存储能力

建议支持：

1. 本地自动保存
2. 手动保存草稿
3. 导出 PDF
4. 导出 PNG
5. 导入 JSON
6. 导出 JSON
7. 示例模板一键加载

注意事项：

1. 仅使用浏览器本地存储时，图片资源可能无法长期稳定保存
2. 如果 Logo 与头像要完整持久化，建议转成 Base64 或接入对象存储
3. PDF 导出必须保证分页稳定，不错位

### 5.5 可用性与细节体验

建议支持：

1. 输入框占位提示
2. 模块示例文案
3. 表单校验
4. 必填项提醒
5. 一键折叠全部
6. 一键展开全部
7. 撤销 / 重做
8. 字数统计
9. 导出前超页提醒
10. 移动端基础可读

## 7. 推荐的技术实现方案

当前项目是 Vue 3 + Vite，适合继续往下搭建。

建议的实现方式：

1. 使用 Vue 3 组合式 API 管理编辑状态
2. 使用组件拆分每个模块，例如 `ResumeProfileForm`、`ResumeInternshipForm`、`ResumePreview`
3. 使用统一状态源管理简历数据
4. 使用本地存储进行草稿保存
5. 使用打印样式或前端导出库生成 PDF

建议的组件结构：

1. `src/components/editor/`
2. `src/components/preview/`
3. `src/components/common/`
4. `src/composables/`
5. `src/data/`
6. `src/utils/`

推荐拆分：

1. 编辑器容器组件
2. 模块表单组件
3. 模块排序组件
4. 预览渲染组件
5. 导出服务
6. 本地存储服务

## 8. 开发优先级建议

为了避免一开始做太大，建议分三期。

### 第一期：MVP 可用版

目标：先做出能填写、能预览、能导出的版本。

必须完成：

1. 基本信息
2. 教育背景
3. 技能模块
4. 实习经历
5. 项目经历
6. 关键词手动加粗
7. 公司 Logo 上传与展示
8. 左右双栏布局
9. 本地保存
10. PDF 导出

### 第二期：体验增强版

目标：让产品真正好用。

建议完成：

1. 模块拖拽排序
2. 模板主题切换
3. JSON 导入导出
4. 字数统计
5. 自动分页提醒
6. 折叠面板优化
7. 一键示例数据

### 第三期：高级增强版

目标：做成完整作品级工具。

建议完成：

1. 富文本工具栏
2. 关键词自动识别加粗
3. 多模板系统
4. 多份简历草稿管理
5. 中英文双语简历
6. 云端存储
7. AI 文案优化辅助

## 9. 验收标准

可以用下面这些标准判断功能是否完成：

1. 用户能在 10 分钟内完成一份基础简历录入
2. 用户可以明确地将关键词设置为加粗
3. 实习经历能稳定展示公司 Logo，且导出后不模糊
4. 预览区与导出结果高度一致
5. 页面看起来专业、整洁，不像普通后台表单
6. 模块多时仍然可以快速定位和编辑
7. 在一页 A4 内的排版可控

## 10. 你现在最值得优先做的功能

结合你的关注点，建议你先按下面顺序做：

1. 搭建左侧编辑区 + 右侧 A4 预览区的整体框架
2. 先完成基本信息、技能、实习经历、项目经历四个核心模块
3. 优先实现“关键词加粗”能力
4. 优先实现“实习经历 Logo 上传与展示”
5. 做一套默认高颜值模板，先把整体观感拉起来
6. 再补本地保存、导出、排序、主题切换

如果只从“最有价值”角度出发，第一批必须做好的就是：

1. 关键词加粗
2. 公司 Logo 展示
3. A4 实时预览
4. PDF 导出
5. 模块化编辑体验

## 11. 后续落地建议

你后面正式开发时，建议直接按下面顺序推进：

1. 先定义统一的简历数据结构
2. 再做编辑器表单
3. 再做预览组件
4. 再做关键词样式解析
5. 再做 Logo 上传与渲染
6. 最后处理导出与主题系统

这样做的好处是：

1. 后续新增模块不会乱
2. 预览和导出会更一致
3. 功能扩展时不用反复推翻结构
